## Abstract

ERC-7756 specifies an on-chain registry for QUIC/HTTP3 agent transport endpoints. Agents register by staking a bond in ETH or ERC-20 tokens; **1 % of the bond is collected as a protocol fee** at registration time via the standardised `X-Opacus-QUIC` signalling header convention. Consumers discover peers by latency budget, transport mode, or kernel-bypass capability, enabling sub-millisecond, physically-routed multi-agent communication over HTTP/3 and eBPF/XDP paths.

## Motivation

As autonomous agents proliferate across decentralised networks, routing decisions must account for transport-layer realities: packet loss, connection setup overhead, and kernel-level latency. HTTP/3 + QUIC eliminates head-of-line blocking and enables 0-RTT connection resumption. eBPF/XDP kernel-bypass further reduces round-trip overhead to 8–20 µs end-to-end. By anchoring transport metadata on-chain, ERC-7756 enables:

* **Trustless endpoint discovery** — any contract or agent can locate the fastest peer without relying on a centralised directory.
* **Transport-mode diversity** — Standard QUIC, eBPF kernel-bypass, and WebTransport datagram modes coexist in a single registry.
* **Latency SLA enforcement** — `latencyBudgetUs` provides an on-chain declaration that task routers can filter by.
* **0-RTT session optimisation** — `zeroRTT` flag enables connection-reuse paths for agent workflows.
* **Protocol fee alignment** — bonded fees create Sybil resistance and fund the Opacus protocol treasury.

## Specification

The key words "MUST", "MUST NOT", "SHOULD", "MAY" in this document are to be interpreted as described in RFC 2119.

| Name | Value | Description |
|---|---|---|
| `QUIC_FEE_BPS` | `100` | Registration fee in basis points (1 %) |
| `OPACUS_TREASURY` | `0xA943F46eE5f977067f07565CF1d31A10B68D7718` | Fee recipient |

### HTTP Header Convention

Compliant agents MUST set the following HTTP/3 response header when communicating via QUIC:

```
X-Opacus-QUIC: mode=<standard|kernel-bypass|datagram>; zero-rtt=<0|1>; latency-us=<us>
```

Example:
```
X-Opacus-QUIC: mode=kernel-bypass; zero-rtt=1; latency-us=12
```

### Enums

```solidity
enum TransportMode {
    Standard,      // Regular QUIC / HTTP3 (Quinn, msquic, etc.)
    KernelBypass,  // eBPF + XDP zero-copy, sub-20 µs end-to-end
    Datagram       // WebTransport datagram mode (browser-compatible)
}
```

### Structs

```solidity
struct Endpoint {
    address       agent;
    string        quicEndpoint;      // e.g. "quic://agent.example.com:4433"
    string        grpcEndpoint;      // e.g. "https://agent.example.com:443" (gRPC-H3)
    TransportMode mode;
    bool          zeroRTT;           // supports QUIC 0-RTT early data
    bool          kernelBypass;      // supports eBPF/XDP kernel-bypass path
    uint32        latencyBudgetUs;   // declared SLA in microseconds (0 = best-effort)
    uint64        registeredAt;
    uint64        updatedAt;
}
```

### Methods

#### `registerEndpoint`

```solidity
function registerEndpoint(
    string  calldata quicEndpoint,
    string  calldata grpcEndpoint,
    TransportMode    mode,
    bool             zeroRTT,
    bool             kernelBypass,
    uint32           latencyBudgetUs,
    address          token,
    uint256          grossAmount
) external payable;
```

MUST collect `grossAmount × 1 %` as fee and forward to `OPACUS_TREASURY` atomically.  
MUST lock remaining 99 % as a recoverable bond.  
MUST revert if `quicEndpoint` is empty.  
MUST revert if the caller is already registered.  
MUST emit `EndpointRegistered(agent, quicEndpoint, mode, kernelBypass)`.

#### `updateEndpoint`

```solidity
function updateEndpoint(
    string  calldata quicEndpoint,
    string  calldata grpcEndpoint,
    TransportMode    mode,
    bool             zeroRTT,
    bool             kernelBypass,
    uint32           latencyBudgetUs
) external;
```

MUST only be callable by the registered agent (`msg.sender` matches stored record).  
MUST NOT collect any fee.  
MUST emit `EndpointUpdated(agent, quicEndpoint, mode)`.

#### `deregisterEndpoint`

```solidity
function deregisterEndpoint() external;
```

MUST release the locked 99 % bond back to `msg.sender`.  
MUST delete the endpoint record.  
MUST emit `EndpointDeregistered(agent)`.  
MUST use a reentrancy guard.

#### `getEndpoint`

```solidity
function getEndpoint(address agent) external view returns (Endpoint memory);
```

MUST return the full `Endpoint` struct for the given agent.

#### `getKernelBypassAgents`

```solidity
function getKernelBypassAgents(uint256 offset, uint256 limit)
    external view returns (address[] memory agents, uint256 total);
```

MUST return agents where `kernelBypass == true`, paginated by `offset`/`limit`.

#### `discoverAgents`

```solidity
function discoverAgents(
    bool   kernelBypassOnly,
    uint32 maxLatencyUs,
    uint256 offset,
    uint256 limit
) external view returns (Endpoint[] memory endpoints, uint256 total);
```

MUST filter by `kernelBypass` if `kernelBypassOnly == true`.  
MUST filter by `latencyBudgetUs <= maxLatencyUs` when `maxLatencyUs > 0`.  
`offset` / `limit` provide pagination.

## Rationale

* **Separate quic + grpc endpoints** — Some agents expose both a raw QUIC socket and a gRPC-over-H3 multiplexed socket; keeping them distinct avoids client-side parsing.
* **latencyBudgetUs as uint32** — 32 bits covers 0–4 294 seconds, sufficient for any real-world SLA while saving storage vs `uint256`.
* **Bond-not-stake** — Fully refundable (minus 1 % fee) to avoid liquidity lockup while still providing Sybil resistance.
* **kernelBypass flag** — eBPF/XDP requires privileged OS access; the flag allows sub-20 µs capable peers to self-identify for MEV bots and arbitrage agents with strict latency requirements.
* **Header convention** — The `X-Opacus-QUIC` header enables HTTP middleware and proxies to enforce transport policies without querying the chain on every request.

## Backwards Compatibility

No existing ERC is modified. ERC-7756 is composable with ERC-7751 (`AgentRecord.quicEndpoint` SHOULD reference an ERC-7756-registered endpoint).

## Security Considerations

* **Reentrancy** — `deregisterEndpoint` releases ETH/ERC-20 and MUST be protected by a reentrancy guard.
* **Endpoint validation** — `quicEndpoint` is stored as an opaque string; consumers MUST validate the URI format off-chain before connecting.
* **Kernel-bypass privilege** — A `kernelBypass=true` claim is self-reported and cannot be verified on-chain. Consumers SHOULD perform a live latency probe before routing critical tasks to self-declared kernel-bypass agents.
* **0-RTT replay** — Agents that set `zeroRTT=true` MUST implement replay protection (e.g., Ticket Age, Anti-Replay Cache) at the transport layer as specified in RFC 9001 §8.
